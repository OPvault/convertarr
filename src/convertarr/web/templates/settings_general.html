Server

Use 0.0.0.0:8000 to expose Convertarr to the LAN. 127.0.0.1:8000 keeps it loopback-only. Only applies when launched via the convertarr command (e.g. convertarr or convertarr --reload). Running uvicorn directly bypasses this — uvicorn's host/port flags win because it binds the socket itself before our code runs.
