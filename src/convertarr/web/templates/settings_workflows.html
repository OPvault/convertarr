How workflows work
Each workflow is a chain of blocks: a trigger, conditions (ANDed), and target codecs. Files are matched in priority order — the lowest priority number wins. When a workflow matches, its target codecs replace the global codec policy for that file. Picking copy for a target means "match this file but leave that track alone".
